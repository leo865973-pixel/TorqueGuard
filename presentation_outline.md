# 扭力扳手校驗系統 (TorqueGuard) - 專案提報綱要

> 本報告專為高階主管會議設計，以「**問題→解方→效益→未來**」為主軸，
> 呈現 AI 輔助開發帶來的典範轉移，與系統上線後可量化的管理效益。

---

## 壹、現場痛點與數位化動機 (Problem Statement)

在傳統作業模式下，扭力扳手校驗管理長期面臨以下挑戰：

| 痛點 | 具體風險 |
|------|---------|
| 📋 紙本記錄易遺失、難追溯 | 品質稽核無法及時調閱歷史數據 |
| ✍️ 人工抄寫數值，易發生筆誤 | 校驗數值不準確，影響生產品質 |
| ⏰ 週期到期無主動提醒機制 | 超期未校驗設備流入生產線 |
| 🗣️ 跨語系操作困難 (外籍移工) | 操作錯誤率高，培訓成本大 |
| 📊 管理者無法即時掌握完成率 | 管理盲點多，介入時機往往已過 |

## 貳、開發模式變革：AI 賦能實證 (AI Case Study)

> [!IMPORTANT]
> 本專案最大的突破在於**開發模式的典範轉移**，採用「AI 結對編程 (Vibe Coding)」：
> **由領域專家主導需求，AI 即時產出程式碼**，打破傳統 IT 開發瓶頸。

### 傳統模式 vs. AI 輔助開發對比

| 指標 | 傳統 IT 委外/開發 | AI 輔助自主開發 |
|------|-----------------|----------------|
| 需求到上線周期 | 數月～數年 | **數天～數週** |
| 需求傳達失真風險 | 高（多層轉譯） | **極低**（直接協作） |
| 功能迭代速度 | 慢（需排期） | **即時回應** |
| 開發與維護成本 | 高（外包費用） | **近乎為零** |
| 深度客製化彈性 | 受限（標準套件） | **完全依現場需求打造** |

**核心價值：** 業務單位不再需要等待 IT，**第一線的防呆邏輯、跨語系需求、現場變通性**，全部由最懂業務的人主導，AI 即時實現。

## 參、系統重點功能 (Core Features)

本系統不僅是記錄工具，更是具備「**防呆、自動追蹤、主動告警、高效管理**」的數位化生產力平台：

> [!IMPORTANT]
> ### 功能一：硬體無縫整合，杜絕人工抄寫錯誤 (RS232 Auto-Capture)
> * **OT/IT 直接融合**：透過瀏覽器 Web Serial API 直接與 RS232 校驗器連線，數值**自動擷取寫入**，從根源消滅人工填寫錯誤。
> * **自動判讀合格/不合格 (PASS / NG)**：系統即時與設備規格比對，超差立即標示 NG 並記錄，無需人工判斷。
> * **斷線彈性**：網路或設備斷線時保留「手動輸入」與完整的「修改履歷追蹤」，兼顧防呆與現場彈性。

> [!IMPORTANT]
> ### 功能二：多層次週期卡控與主動到期預警
> * **自動週期卡控**：系統自動計算每把扳手的校驗週期與當日次數額度，超期或超額時強制攔截，**設備無法被誤用**。
> * **到期預警橫幅**：儀表板頂部即時顯示「即將到期或已超期」的設備清單，讓管理者**第一時間掌握異常**，化被動為主動。
> * **Progress Dashboard**：首頁即時顯示當日/當週校驗完成率進度條，管理者進門一眼掌握全局。

> [!TIP]
> ### 功能三：自動化主動通報，實現閉環管理 (Email Alert)
> * **被動轉主動**：系統整合雲端排程，針對「週期未完成校驗」的設備，**自動定期發送 Email 通知**給相關主管與單位負責人。
> * **閉環追蹤**：確保沒有任何校驗任務遺漏，將人為疏忽風險降至最低。

> [!TIP]
> ### 功能四：全方位追溯與一鍵報表 (Traceability & Reporting)
> * **多維度歷史查詢**：依日期範圍、設備代碼、操作員、校驗結果任意組合篩選，品質稽核「一秒調閱」所有歷史履歷。
> * **一鍵 PDF 標準報表**：整合數據統計與趨勢折線圖，複雜數據一鍵匯出清晰 PDF，讓主管快速掌握品質動態。
> * **系統操作日誌**：完整記錄所有後台設定異動（誰、何時、改了什麼），杜絕資料被竄改後難以追責的問題。

> [!TIP]
> ### 功能五：Excel 批次建檔與即時管理
> * **批次匯入**：針對動輒數百把設備清單，後台支援一鍵匯入 Excel 範本，取代傳統逐筆建檔的曠日費時。
> * **即時線上編輯**：管理員能即時修改規格，並支援批次「停用/啟用」，大幅降低龐大基礎資料的維護成本。

> [!NOTE]
> ### 功能六：零門檻多語系設計 (中 / 泰 雙語)
> * 登入介面與主操作介面支援**一鍵即時切換中文/泰文**，解決外籍移工操作困難問題，降低現場培訓成本。
> * 採用直覺的卡片式 UI 與 Apple 設計美學，介面清晰易懂，上手時間極短。

## 肆、部署與擴展優勢 (Deployment & Scalability)

> [!TIP]
> ### 極致輕量，隨插即用 (Zero-Backend Architecture)
> * **無需安裝伺服器或資料庫**：系統主體為單一 HTML 檔案，搭配 Google Firebase 雲端資料庫。
> * 只要有**瀏覽器與網路**即可上線，硬體門檻與部署成本趨近於零。
> * 資料儲存於雲端，天然具備多設備同步與備份特性。

> [!IMPORTANT]
> ### 水平複製，快速推廣 (Horizontal Scalability)
> * **多單位獨立架構已內建**：後台以「單位代碼」隔離各廠區/部門資料，無需重新開發。
> * 推廣至新廠區只需：**後台新增單位代碼 → 匯入設備清單 → 分配帳號**，即可獨立運作。
> * 可快速作為公司其他**治具、量具、量測設備**數位化管理的標準範本。

> [!NOTE]
> ### 高韌性離線備援 (High Resilience)
> * 系統具備**本機資料暫存**機制，網路斷線時仍可繼續操作，恢復後**自動同步雲端**。
> * 確保生產線校驗作業不因網路問題中斷。

## 陸、總結與展望 (Conclusion)

本專案成功驗證了「**業務單位 × AI 自主解決問題**」的可行性：

1. **短期效益**：徹底解決現場扭力扳手校驗的管理痛點，實現數位化、可追溯、零遺漏。
2. **中期效益**：本套架構可作為公司**量具/治具數位化管理的標準範本**，快速水平複製到各大廠區。
3. **長期效益**：驗證了 AI 輔助開發模式，未來類似的**輕量級生產力工具**，業務單位皆可自主快速實現，大幅縮短從「需求」到「上線」的距離。

> **核心訊息：這不只是一套校驗系統，更是「AI 賦能製造現場」的實證案例。**
